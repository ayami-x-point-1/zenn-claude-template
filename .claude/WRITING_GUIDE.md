# 執筆ガイドライン

このファイルには、技術ブログを執筆する際の基本方針を記載します。

## 文体・スタイル

### 基本方針
- **ですます調**で統一
- 技術的な正確性を最優先
- 比喩を使わず、ロジックと事実のみで説明
- 一言で簡潔にまとめてから詳細を説明

### 構成
1. **導入**：記事の目的を1-2文で簡潔に説明
2. **本文**：ロジカルに段階的に説明
3. **まとめ**：学んだことや次のステップを提示

## 技術レベル

### 想定読者
- 基礎知識がある中級エンジニア
- 特定の技術スタックに依存しない普遍的な知識を求める読者

### 説明の深さ
- 前提知識は明記する
- 用語の定義は初出時に記載
- コード例は最小限かつ動作可能なものを提示

## コード記述

### コードブロック
- 言語を明示（```typescript、```python など）
- コメントは必要最小限
- 実行可能なコードを優先

### サンプルコード
- 宣言的プログラミングスタイル
- 関数型の思考で記述
- オブジェクト指向は必要な場合のみ

## 避けるべき表現

- 曖昧な表現（「〜のようなもの」「だいたい〜」）
- 比喩による説明（「郵便局で例えると〜」など）
- 感情的な表現（「すごい」「便利」などの形容詞を多用しない）

## 推奨する表現

- 事実ベースの説明
- ロジックの流れを明確に
- 「この関数は〇〇をするための関数です」といった明確な定義

## 記事の種類

### 技術解説
- 特定の技術や概念を深く掘り下げる
- 内部動作やアーキテクチャを説明

### チュートリアル
- 手順を明確に記載
- 各ステップの目的を説明
- 最終的な成果物を明示

### トラブルシューティング
- 問題の症状を明確に記載
- 原因の特定方法を説明
- 解決策と予防策を提示

## 記事のメタデータ

### タイトルの決め方

#### タイトルの基本原則
1. **具体性**: 何について書かれているかが明確
2. **簡潔性**: 50文字以内を目安
3. **検索可能性**: 検索されそうなキーワードを含む
4. **ターゲット明確**: 誰に向けた記事かが分かる

#### 記事種類別のタイトルテンプレート

**技術解説の場合**
- `[技術名]の[特定の概念]を理解する`
- `[技術名]の内部動作を解説`
- `[誤解されがちな概念]の本当の意味`
- `[技術A]と[技術B]の違いを理解する`

例:
- ✅ サーバーレスアーキテクチャの本当の意味 - 「サーバーがない」わけではない
- ✅ TanStack RouterとReact Routerの違いを理解する
- ❌ サーバーレスについて（抽象的すぎる）

**チュートリアルの場合**
- `[技術名]で[成果物]を作る`
- `[技術名]の実装ガイド - [具体的な機能]編`
- `[技術名]入門 - [ゴール]まで`
- `ゼロから始める[技術名]実装`

例:
- ✅ TanStack Routerで型安全なルーティングを実装する
- ✅ DynamoDB入門 - RDBMSエンジニアのためのNoSQL設計ガイド
- ❌ TanStack Routerを使ってみた（成果物が不明瞭）

**トラブルシューティングの場合**
- `[技術名]で[エラー]が出たときの対処法`
- `[技術名]実装時のハマりポイントと解決策`
- `[エラーメッセージ]の原因と対策`
- `[技術名]のよくあるエラー[N]選`

例:
- ✅ Gemini APIのエラーメッセージに惑わされないために - 実装時のハマりポイント3選
- ✅ "The document has no pages"エラーの本当の原因 - Gemini API実装ガイド
- ❌ Gemini APIで困ったこと（問題が不明瞭）

#### 効果的なタイトルの要素

**含めるべき要素**
- 技術名（Gemini API, TanStack Router, DynamoDBなど）
- 具体的な問題や成果物
- ターゲット読者（初学者、RDBMSエンジニアなど）

**避けるべき要素**
- 曖昧な表現（「〜してみた」「〜について」）
- 感情的な表現（「すごい」「便利」）
- 釣りタイトル（内容と乖離したタイトル）

### トピックの選び方

#### トピックとは
Zennの「トピック」は記事を分類するためのタグです。読者が関連記事を見つけやすくなります。

#### トピックの命名規則（重要）

Zennのトピックには**入力ルール**があります：

**基本ルール**
- **小文字で入力**: `typescript`（✅）、`TypeScript`（❌）
- **記号を除く**: `nextjs`（✅）、`next.js`（❌）
- **日本語はそのまま**: `エラーハンドリング`（✅）

**具体例**

| 技術名 | 入力するトピック名 | 表示名 |
|--------|-------------------|--------|
| Next.js | `nextjs` | Next.js |
| TypeScript | `typescript` | TypeScript |
| React | `react` | React |
| TanStack Router | `tanstackrouter` | TanStack Router |
| Gemini API | `gemini` または `geminiapi` | Gemini / Gemini API |
| DynamoDB | `dynamodb` | DynamoDB |
| AWS | `aws` | AWS |
| エラーハンドリング | `エラーハンドリング` | エラーハンドリング |

**確認方法**
既存のトピックを使う場合は、Zennのトピックページ（例: https://zenn.dev/topics/nextjs）で「このトピックを指定するには〇〇と入力」という説明を確認してください。

#### 選定基準
1. **メイン技術**: 記事の中心となる技術を1-2個選ぶ
2. **関連技術**: 使用している言語やフレームワークを1-2個選ぶ
3. **記事の性質**: エラーハンドリング、トラブルシューティングなど

#### トピックの数
- **推奨**: 3-5個
- **最低**: 2個
- **最大**: 5個

#### トピック選定例

**Gemini API記事の場合**
```yaml
topics: ["gemini", "typescript", "エラーハンドリング", "googleai"]
```
- `gemini`（メイン技術）
- `typescript`（使用言語）
- `エラーハンドリング`（記事の性質）
- `googleai`（関連技術、または`ai`）

**TanStack Router記事の場合**
```yaml
topics: ["tanstackrouter", "react", "typescript", "ルーティング"]
```
- `tanstackrouter`（メイン技術）
- `react`（関連フレームワーク）
- `typescript`（使用言語）
- `ルーティング`（記事のテーマ）

**DynamoDB記事の場合**
```yaml
topics: ["dynamodb", "aws", "nosql", "typescript"]
```
- `dynamodb`（メイン技術）
- `aws`（プラットフォーム）
- `nosql`（データベース種類）
- `typescript`（使用言語）

#### トピック名の検証

新しいトピックを作る前に：
1. Zennで既存トピックを検索（例: https://zenn.dev/topics/トピック名）
2. 既存のトピックがあれば、そのトピック名を使う
3. 新規トピックを作る場合は、小文字・記号なしのルールに従う

### 絵文字の選び方

#### 絵文字の役割
記事の内容を視覚的に表現し、記事一覧での識別性を高めます。

#### 記事種類別の推奨絵文字

**技術解説**
- 📚 書籍（知識、学習）
- 🔍 虫眼鏡（調査、深堀り）
- 💡 電球（理解、気づき）
- 🧩 パズル（仕組み、構造）
- 🏗️ 建設（アーキテクチャ）

**チュートリアル**
- 🛠️ 工具（実装、構築）
- 📝 メモ（手順、ガイド）
- 🚀 ロケット（開始、導入）
- ⚙️ 歯車（設定、構成）
- 🎯 的（ゴール、目標）

**トラブルシューティング**
- 🐛 虫（バグ、エラー）
- 🔧 レンチ（修正、解決）
- ⚠️ 警告（注意、問題）
- 🔥 炎（緊急、重要）
- 🩹 絆創膏（対処、修復）

#### 技術別の推奨絵文字

**AI・API関連**
- 🤖 ロボット（AI、自動化）
- 🔌 プラグ（API、連携）
- 💬 吹き出し（チャット、対話）

**データベース関連**
- 🗄️ キャビネット（データベース）
- 💾 フロッピー（データ、保存）
- 🔑 鍵（キー、認証）

**フロントエンド関連**
- 🎨 パレット（UI、デザイン）
- 🖼️ 額縁（コンポーネント）
- 🌐 地球（Web、ブラウザ）

#### 絵文字選定のポイント
- 記事の**メインテーマ**を表現する
- 記事の**種類**（解説、チュートリアル、トラブルシューティング）を考慮
- 視認性の高いシンプルな絵文字を選ぶ
- 複数の意味を持つ絵文字は避ける

#### 選定例

**Gemini API トラブルシューティング記事**
- 🐛（バグ・エラー） - 推奨
- 🔧（修正・解決） - 推奨
- ⚠️（警告・注意） - 推奨
- 🤖（AI関連） - OK
- 📚（解説系） - 記事種類と不一致

**TanStack Router チュートリアル記事**
- 🛠️（実装・構築） - 推奨
- 🚀（開始・導入） - 推奨
- 🗺️（ルーティング） - 推奨
- 🐛（エラー系） - 記事種類と不一致

**DynamoDB 技術解説記事**
- 🗄️（データベース） - 推奨
- 🔍（調査・深堀り） - 推奨
- 💡（理解・気づき） - 推奨
- 🛠️（実装系） - 記事種類と不一致
