記事をレビューして公開前の確認を行います。

## レビュー項目

### 1. メタデータの確認
- title: 適切なタイトルか
- emoji: 記事内容に合っているか
- type: tech または idea が適切に選択されているか
- topics: 5つ以内で設定されているか
- published: 公開設定が正しいか

### 2. 内容の確認
- 誤字脱字がないか
- コードブロックに言語指定があるか
- コードは動作可能か
- リンクは正しく機能するか
- 画像は適切に表示されるか

### 3. 執筆方針の確認（`WRITING_GUIDE.md`に基づく）
- 比喩を使わず事実ベースで説明しているか
- 一言でまとめてから詳細を説明しているか
- ですます調で統一されているか
- 曖昧な表現を避けているか

### 4. 構成の確認
- 導入部分で記事の目的が明確か
- 本文は論理的に構成されているか
- まとめ部分があるか
- 見出しの階層が適切か

### 5. 技術的正確性
- 技術用語の定義が正確か
- 前提知識が明記されているか
- コード例が最小限で理解しやすいか

## レビュー手順

1. **記事ファイルを読む**
   - `articles/`ディレクトリ内の対象ファイルを確認

2. **チェックリストに沿って確認**
   - 上記のレビュー項目を順番に確認

3. **改善点を指摘**
   - 問題がある箇所を具体的に指摘
   - 修正案を提示

4. **プレビューの確認を推奨**
   - `npx zenn preview` でブラウザ表示を確認するよう促す

## 出力形式

レビュー結果を以下の形式で出力：

```
# レビュー結果

## 良い点
- ...

## 改善が必要な点
- [ ] 項目1（ファイルパス:行番号）
- [ ] 項目2（ファイルパス:行番号）

## 推奨事項
- ...
```
