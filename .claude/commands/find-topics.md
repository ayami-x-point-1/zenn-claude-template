記事のトピックを発見するためのサポートを行います。

## 手順

### 1. 興味・経験の確認

以下の質問を通じて記事トピックの候補を探します：

**最近の技術的な経験**
- 最近取り組んだプロジェクトや実装はありますか？
- 解決した技術的な問題はありますか？
- 新しく学んだ技術やツールはありますか？

**日常の開発での気づき**
- 開発中に「これは他の人にも役立つかも」と思ったことはありますか？
- ドキュメントが不足していると感じた技術はありますか？
- チームメンバーによく質問される内容はありますか？

**興味のある技術領域**
- 深く学びたい技術はありますか？
- 最近気になっているトレンドや技術はありますか？
- 専門性を高めたい分野はありますか？

### 2. `LEARNING_LOG.md` の確認

日々の学習メモから記事化できるトピックを探します。

### 3. `TOPIC_DISCOVERY.md` の活用

トピック発見のフレームワークに沿って、記事アイデアを生成します。

### 4. トピック候補の提案

以下の観点でトピック候補を複数提案します：
- **実践的な価値**: 読者が実際に使える内容か
- **独自性**: 他の記事との差別化ポイント
- **執筆しやすさ**: すでに持っている知識・経験で書けるか
- **需要**: 読者の関心が高そうか

### 5. `ARTICLE_IDEAS.md` への記録

決定したトピックを「アイデア一覧」に追加します。

## 出力形式

```markdown
# トピック候補

## 候補1: [トピック名]
- **概要**: ...
- **想定読者**: ...
- **執筆しやすさ**: ★★★☆☆
- **実践的価値**: ★★★★☆
- **独自性**: ★★★☆☆

## 候補2: [トピック名]
...

## 推奨トピック
最も書きやすく価値の高いトピックは「〇〇」です。
理由: ...
```

## 次のステップ

トピックが決まったら `/new-article` で記事作成を開始できます。
