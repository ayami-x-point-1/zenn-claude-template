# 投稿ワークフロー

このファイルには、記事を公開するまでの手順とチェックリストを記載します。

## 記事作成の流れ

### 1. 記事の作成
```bash
npx zenn new:article
```
または、スラッシュコマンド `/new-article` を使用

### 2. 執筆
- テンプレートに沿って執筆
- `WRITING_GUIDE.md` の方針に従う
- コードは動作確認済みのものを使用

### 3. プレビュー確認
```bash
npx zenn preview
```
- ブラウザで http://localhost:8000 を開く
- レイアウトの確認
- コードブロックの表示確認
- 画像の表示確認

### 4. レビュー
スラッシュコマンド `/review-article` を使用してレビュー実施

## 投稿前チェックリスト

### メタデータ
- [ ] `title`: 適切なタイトルが設定されているか
- [ ] `emoji`: 記事内容に合った絵文字が設定されているか
- [ ] `type`: `tech` または `idea` が適切に選択されているか
- [ ] `topics`: 関連するトピックが5つ以内で設定されているか
- [ ] `published`: 公開する場合は `true` に設定

### 内容
- [ ] 誤字脱字がないか
- [ ] コードブロックに言語が指定されているか
- [ ] コードは動作確認済みか
- [ ] リンクは正しく機能しているか
- [ ] 画像は適切に表示されているか
- [ ] 比喩を使わず事実ベースで説明しているか
- [ ] 一言でまとめてから詳細を説明しているか

### 構成
- [ ] 導入部分で記事の目的が明確か
- [ ] 本文は論理的に構成されているか
- [ ] まとめ部分があるか
- [ ] 見出しの階層が適切か

### 技術的正確性
- [ ] 技術用語の定義が正確か
- [ ] 前提知識が明記されているか
- [ ] コード例が最小限で理解しやすいか

## GitHub連携での公開

### 初回セットアップ
1. GitHubリポジトリを作成
2. Zennと連携設定
3. ローカルリポジトリをGitHubにpush

### 記事の公開
1. 記事のメタデータで `published: true` に設定
2. Gitにコミット
3. GitHubにpush
4. Zennに自動反映される

```bash
git add articles/
git commit -m "記事を追加: [タイトル]"
git push
```

## 下書き保存

下書きとして保存する場合：
- メタデータで `published: false` のまま
- GitHubにpushしても非公開のまま
- Zennのダッシュボードで確認可能

## 記事の更新

1. ローカルでMarkdownファイルを編集
2. プレビューで確認
3. Gitにコミット＆push
4. Zennに自動反映

## トラブルシューティング

### プレビューが表示されない
- `npx zenn preview` が正しく実行されているか確認
- ポート8000が他のプロセスで使用されていないか確認

### 記事が反映されない
- GitHubとZennの連携設定を確認
- GitHubにpushされているか確認
- Zennのダッシュボードでエラーがないか確認
